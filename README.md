# 🐎 Hippodrome Management System

## 📖 Описание проекта

**Hippodrome Management System** — это консольное приложение на языке C для управления ипподромом. Система позволяет администраторам, жокеям и владельцам лошадей эффективно управлять информацией о лошадях, жокеях, забегах и результатах соревнований.

### Основные возможности

- **🔐 Аутентификация и авторизация** — три роли пользователей: администратор, жокей, владелец
- **🐴 Управление лошадьми** — CRUD операции с информацией о лошадях
- **🏇 Управление жокеями** — полная информация о жокеях и их достижениях
- **👥 Управление владельцами** — данные о владельцах лошадей
- **🏁 Управление забегами** — создание забегов, добавление участников, фиксация результатов
- **💰 Распределение призов** — автоматическое распределение призового фонда (50%/30%/20%)
- **📊 Отчёты** — разнообразные отчёты для каждой роли пользователя
- **💾 Хранение данных** — SQLite база данных с поддержкой транзакций

### Вариант задания №4 — «Ипподром»

Проект выполнен в рамках лабораторной работы №4 по курсу "Технологии программирования для мобильных приложений". Реализованы все требования согласно варианту 4 «Ипподром».

## 🚀 Установка и запуск

### Требования

- **Компилятор GCC** (версия 4.8 или выше)
- **SQLite3** (версия 3.0 или выше)
- **Make** (для автоматизации сборки)

### Установка зависимостей

### Сборка проекта

1. **Клонирование репозитория**
```bash
git clone https://github.com/yourusername/hippodrome.git
cd hippodrome
```

2. **Сборка приложения**
```bash
make clean
make all
```

3. **Запуск приложения**
```bash
make run
```

Или напрямую:
```bash
./bin/ippodrom
```

### Очистка

```bash
# Удаление бинарных файлов
make clean

# Полный сброс (удаление базы данных и конфигурации)
make reset
```

## 📋 Использование

### Первый запуск

1. При первом запуске приложение автоматически создаст:
   - Базу данных `hippodrome.db`
   - Файл конфигурации `hippodrome.conf`
   - Все необходимые таблицы

2. **Создание администратора:**
   - Выберите опцию `2. Register (Admin only)`
   - Будет создан пользователь `admin` с паролем `admin`

3. **Вход в систему:**
   - Выберите опцию `1. Login`
   - Username: `admin`
   - Password: `admin`

## 📊 Отчёты

### Для администратора
- Список всех лошадей
- Список всех жокеев
- Список всех владельцев
- Полная история забегов
- Статистика по забегам за период

### Для жокея (*)
- История участия в забегах
- Личная статистика (победы, топ-3)
- Заработок по периодам

### Для владельца (**)
- Список лошадей с результатами забегов
- Лошадь с максимальным количеством побед
- Статистика по каждой лошади

## 🗄️ Структура базы данных

```sql
-- Пользователи системы
USERS(id, username, password, role)

-- Владельцы
OWNERS(id, name, last_name, middle_name, address, phone, user_id)

-- Жокеи
JOCKEYS(id, last_name, experience_years, birth_year, address, user_id)

-- Лошади
HORSES(id, nickname, age, experience_years, owner_id)

-- Забеги
RACES(id, race_date, race_number, prize_fund)

-- Участники забегов
RACES_HORSES_JOCKEYS(id, race_id, horse_id, jockey_id, position)

-- Распределение призов
PRIZE_DISTRIBUTION(id, race_id, horse_id, position, prize_amount, distribution_date)
```
